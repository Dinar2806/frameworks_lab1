# Лабораторная работа №1: Анализ данных и машинное обучение на Python
**Тема:** Прогнозирование успеваемости (CGPA) и риска депрессии у студентов

## Описание задачи

Цель работы — провести полный цикл анализа данных (EDA), предобработки и построения моделей машинного обучения для решения двух задач:
1.  **Регрессия:** Прогнозирование среднего балла (CGPA) студента на основе его режима сна, учебы и уровня стресса.
2.  **Классификация:** Определение наличия риска развития депрессии (`Depression`) на основе образа жизни и академической успеваемости.

## Используемые технологии

*   **Язык:** Python 3.x
*   **Обработка данных:** `pandas`, `numpy`
*   **Визуализация:** `matplotlib`, `seaborn`
*   **Машинное обучение:** `scikit-learn` (LinearRegression, LogisticRegression, StandardScaler, train_test_split)

## Структура проекта

*   `lab_1.ipynb` — основной ноутбук с кодом и визуализациями.
*   `data/dataset.csv` — исходный набор данных.
*   `README.md` — описание проекта.

## Этапы выполнения работы

### 1. Разведочный анализ данных (EDA)
*   Загрузка данных и проверка на пропуски (`isnull().sum()`).
*   Визуализация распределения целевых переменных:
    *   Гистограмма CGPA (нормальное/скошенное распределение).
    *   Столбчатая диаграмма Depression (проверка баланса классов).
*   Построение корреляционной матрицы (Heatmap) для выявления линейных связей между признаками.
- Вывод: пропуски и дубликаты не найдены, распределение по Гауссу

### 2. Предобработка данных
*   **Очистка:** исправление логических ошибок (отрицательный возраст, CGPA > 10, время суток > 24 часов).
*   **Заполнение пропусков (если бы были в датасете):**
    *   Числовые признаки: медиана (`median`).
    *   Категориальные признаки: мода (`mode`).
* Самый важный этап - кодирование категориальных признаков
### 3. Feature Engineering (Конструирование признаков)
* Выполнено кодирование категориальных признаков
Созданы новые признаки для улучшения качества моделей:
*   `Study_Social_Ratio`: Отношение часов учебы к часам в соцсетях (показатель эффективности использования времени).
*   `Health_Index`: Индекс здоровья, учитывающий физическую активность и уровень стресса.
*   `Adequate_Sleep`: Бинарный признак достаточности сна (7–9 часов).


### 4. Разделение данных и Масштабирование
*   **Схема разделения:** 70% Train / 15% Validation / 15% Test.
*   **Стратификация:** Для задачи классификации использован параметр `stratify`, чтобы сохранить пропорцию классов (депрессия/нет) во всех выборках.
*   **Масштабирование:** `StandardScaler` применен только после разделения данных (fit на Train, transform на Val/Test) для предотвращения утечки данных (Data Leakage).

### 5. Моделирование

#### А. Линейная регрессия (Linear Regression)
*   **Цель:** Предсказать `CGPA`.
*   **Метрики:** $R^2$, MAE, MSE, RMSE.
*   **Диагностика:** Построены кривые обучения и графики остатков (Residuals Plot) для проверки линейности связи.

#### Б. Логистическая регрессия (Logistic Regression)
*   **Цель:** Классифицировать риск депрессии (0/1).
*   **Подбор гиперпараметров:** Эксперименты с коэффициентом регуляризации `C` и числом итераций `max_iter`.
*   **Метрики:** Accuracy, F1-Score, ROC-AUC.
*   **Борьба с переобучением:** Анализ кривых обучения (Learning Curves). Сравнение Accuracy на Train и Test.

### 6. Оценка качества и Выводы

#### Регрессия (CGPA)
*   **Наблюдение:** Низкий коэффициент детерминации ($R^2 \approx 0.02$).
*   **Причина:** Слабая линейная зависимость между образом жизни и оценками в данном датасете. Модель стремится предсказывать среднее значение, игнорируя индивидуальные особенности (высокое стандартное отклонение реальных данных vs низкое у предсказаний).
*   **Рекомендация:** Требуется сбор более релевантных признаков или использование нелинейных моделей.

#### Классификация (Depression)
*   **Проблема дисбаланса классов:** В данных значительно больше студентов без депрессии.
*   **Результат:** Модель склонна предсказывать мажоритарный класс (0), что дает высокую Accuracy, но низкий Recall/F1 для класса "Депрессия".
*   **Решение:** Рекомендуется использовать взвешивание классов или изменение порога классификации (threshold tuning) для повышения чувствительности модели к меньшему классу.

## Ключевые визуализации

1.  **Correlation Heatmap:** Показывает силу линейных связей.
2.  **Residuals Plot (Регрессия):** Проверка случайности ошибок.
3.  **Learning Curves:** Диагностика переобучения/недообучения.
4.  **Class Distribution (Классификация):** Сравнение реального и предсказанного распределения классов.
